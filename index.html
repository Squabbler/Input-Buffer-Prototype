Input Buffer Prototype

This is a prototype to test an input buffer system seen in fighting games, etc. The idea is every update frame, the input is captured and stored. The last X-number (in our example, 10) inputs are saved as a cache in a buffer set to check for move patterns.

Every update frame after the buffer set has been updated with the latest input, it is run against a set of move patterns to check for a match. One of two scenarios will occur:

  • If a pattern is detected, it will fire that move (based on priority) and clear out the buffer set.
  • If no pattern is detected, it will continue to the next frame.

Note: for this demo, the update loop is set as a 100ms interval (frame) to provide a slow motion understanding of what is happening at full speed (30-60 fps)

Inputs

  • w = up
  • s = down
  • a = left
  • d = right
  • p = punch
  • k = kick

Moves (priority order)

  1. Super Punch = s d s d p
  2. Ultra Kick = s a s a k
  3. Fireball = s d p
  4. Spin Kick = s a k
  5. Punch = p
  6. Kick = k
